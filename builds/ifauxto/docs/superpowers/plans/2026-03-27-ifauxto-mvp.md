# iFauxto MVP вЂ” Implementation Plan (Entry Point)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build iFauxto вЂ” a manual-first iOS photo organization app that overlays Apple Photos with custom folders, drag-and-drop ordering, and CloudKit sync.

**Architecture:** Overlay model вЂ” PHAsset references only (no photo duplication). SwiftData for local persistence with CloudKit sync via `ModelConfiguration(.private(...))`. SwiftUI views with native drag-and-drop reordering.

**Tech Stack:** Swift 5.9 В· SwiftUI В· SwiftData В· PhotoKit (PHAsset) В· CloudKit В· iOS 17.0+ В· XcodeGen

## Key Rules

- **Never duplicate photos.** All photo references store `PHAsset.localIdentifier` only.
- **Never auto-reorder.** All ordering is explicit `orderIndex` integers. No sorting by date anywhere in the UI.
- **SwiftData CloudKit:** Use `ModelConfiguration(cloudKitDatabase: .private("iCloud.com.ifauxto.app"))` вЂ” not `NSPersistentCloudKitContainer`.
- **iOS 17+ only.** Use `#Predicate`, `@Model`, SwiftUI `.draggable()` / `.dropDestination()`.
- **XcodeGen generates the xcodeproj.** Never hand-edit the `.xcodeproj` file.
- **Bundle ID:** `com.ifauxto.app` В· CloudKit container: `iCloud.com.ifauxto.app`
- **Development Team:** Leave as `""` in project.yml вЂ” Jordan sets this in Xcode before first build.
