Where should repo-man live?

Pick the home that fits best

A

~/openclaw/repo-man/ ⭐ recommended

Pros

  • Consistent with every other OpenClaw project
  • openclaw_profile.json, .env, and scripts all adjacent
  • Agents/Lobster workflows can reference it by simple relative path
  • Shows up naturally in repo-queue triage

Cons

  • Adds one more root-level dir to openclaw (already has ~20)
B

~/openclaw/projects/repo-man/

Pros

  • Keeps root cleaner if you intend lots of projects
  • Logical grouping under a "projects" namespace

Cons

  • projects/ currently only has projects.json — not a real project dir
  • Breaks the pattern every other project uses
  • Adds an extra path hop for agent references
C

~/repo-man/ (standalone, outside openclaw)

Pros

  • Fully independent — clean git repo, own lifecycle
  • Makes Phase 2 (hosted SaaS) easier to extract and deploy
  • No openclaw dependency if you ever spin it out

Cons

  • openclaw_profile.json needs explicit path config to find openclaw data
  • Agents can't reference it without absolute paths or env vars
  • Friction if you want openclaw crons or Lobster to trigger it
